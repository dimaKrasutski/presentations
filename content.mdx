## Минусы Typescript

<div>
<Appear>
  <li>дополнительная абстракция над javascript</li>
  <li>typescript потребует больше времени для погружения в проект новых людей, которые ранее работали только в JS</li>
  <li>сложные типы могут быть сложными для понимания при интеграции TypeScript в существующую кодовую базу</li>

## Как начать работать с TypeScript

<div>
  <li> установка typescript из npm</li>
  <li> tsconfig </li>
  <li> react + typescript: установка типов @types/node, @types/react, @types/react-dom</li>
  <li> eslint, (tslint - deprecated) </li>
  <li> файлы .ts и .tsx, .d.ts для добавления типов в уже существующие библиотеки JavasScript  </li>
</div>

##### В TypeScript имеются следующие базовые типы:
  <li> Boolean: логическое значение true или false</li>
    <li>Number: числовое значение</li>
    <li>String: строки</li>
    <li>Array: массивы</li>
    <li>Tuple: [123, 'dima']</li>
    <li>Enum: перечисления</li>
    <li>Any: произвольный тип</li>
    <li>Null и undefined: соответствуют значениям null и undefined в javascript</li>
    <li>Array: массивы</li>
    <li>Void: возвращаемый типа функций</li>
    <li>Never: отсутствие значения и используется в качестве возвращаемого типа функций, которые генерируют или возвращают ошибку</li>
